Validé : éléments actés et livrés

DEL‑001 · Suppression d'intervention

Validé Type : Fonctionnel

Ajout d'un bouton supprimer dans le modal d'intervention.

  • Bouton visible dans le modal d'édition d'intervention
  • Confirmation avant suppression définitive
  • Fonction de base demandée par le métier

UX‑001 · Fermeture hors modal

Validé Type : UX

Clic extérieur au modal = fermeture automatique.

  • Standard UX respecté
  • Amélioration de l'expérience utilisateur

MODAL‑001 · Réorganisation champs modal intervention

Validé Type : UX

Ajustement UX des champs dans le modal d'intervention.

  • Nom : champ renommé et repositionné
  • Adresse : séparation du code postal
  • Réagencement des champs pour une meilleure ergonomie

BUG‑001 · Bug rafraîchissement CRM

Corrigé Type : Bug fix

Correction du bug de changement de background color lors du rafraîchissement du CRM.

RSZ‑001 · Resize CRM

Corrigé Type : UI

Correction du niveau de zoom sur le CRM (refonte du legacy).

  • Bon niveau de zoom appliqué par défaut
  • Amélioration de l'affichage responsive

ART‑REL‑001 · Liste « Mes artisans à relancer »

STAT‑001 · Sous-statuts intervention

À faire Type : Métier Inclus

Ajout de sous-catégories aux statuts d'intervention.

  • Prévu dans l'évolution métier
  • Permettra une granularité plus fine dans le suivi

RETARD‑001 · Gestion des retards dans le dashboard

À faire Type : Dashboard Inclus

Amélioration du dashboard pour la gestion des retards.

Fonctionnalités à implémenter

  • Affichage du retard : quel heure de retard ? (seuil : 10h)
  • Enregistrement du nombre de retards dans l'année
  • Notification par mail lors d'un retard
  • Affichage du nombre de retards annuels sur le dashboard
📋 Spécifications :
  • Seuil de retard à définir : 10 heures de dépassement
  • Compteur annuel visible sur le dashboard
  • Système d'alerte email automatique

UI‑002 · Homogénéisation des couleurs

À faire Type : UI Inclus

  • Cohérence visuelle sur toutes les pages

À faire Type : Données Inclus
