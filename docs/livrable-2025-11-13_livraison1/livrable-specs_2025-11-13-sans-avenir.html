1) Tâches effectuées ✅

Validé : éléments actés et livrés

Vue d'ensemble

Validé & livré
  • Règles due date → statut « Check », et retour au statut précédent via modification de la date de fin.
  • Champs obligatoires à la création d'une intervention : Adresse, Contexte, Métier, Statut, Agence.
  • Agences avec Référence agence obligatoire : ImoDirect, AFEDIM, Locoro.

MSG‑003 · Mapping champ « Montant du budget »

Validé Type : Données

Le champ Montant du budget correspond à la colonne « SST » du Google Sheet actuel.

INT‑001 · Champs obligatoires

Validé Type : Données
  • Adresse, Contexte, Métier, Statut, Agence sont requis à la création (blocage si manquant).

INT‑003 · Contexte : droits d'édition

Validé Type : Règle produit
  • Modifiable uniquement à la création (NewInterventionModalContent).
  • InterventionModalContent → en lecture seule.
  • Seuls les Administrateurs peuvent modifier Contexte après création.
📝 Tutoriel de test :
  1. Pour tester avec andrea : créer une intervention, une fois créée aller dans l'édition, il ne peut plus modifier la consigne.
  2. Pour tester avec badr : même test sauf que lui pourra modifier la consigne.

DAT‑001 · Due date dépassée → statut « Check »

Validé Type : Automatisation
  • Si la due date est dépassée et le statut est « Visite technique » ou « Intervention en cours », le statut passe automatiquement à « Check ».
  • Pour revenir au statut précédent (VT / EC), l'utilisateur doit modifier la date de fin (prolongation validée).
  • Les dossiers en VT / EC doivent obligatoirement avoir une due date (blocage à défaut).

AGN‑001 · Référence agence — règle & modèle de données

Validé Type : Données

À la création d'une intervention, la Référence agence est obligatoire pour :

  • ImoDirect
  • AFEDIM
  • Locoro

Le champ Référence agence fait partie du modèle de données (base de données) et est stocké sur l'intervention. Pas de tâche distincte.

ARC‑001 · Archivage avec commentaire obligatoire

Validé Type : Gouvernance
  • À l'archivage d'un artisan ou d'une intervention terminée, saisir un commentaire (bloquant).

ARC‑002 · Indication « Indisponible »

Validé Type : UX
  • Les artisans indisponibles apparaissent dans les recherches, avec une pastille « Indisponible » clairement visible.
  • Les artisans archivés n'apparaissent pas sur la carte.
  • Un artisan archivé conserve son statut de compétence (Novice, Confirmé, Potentiel, etc.).

ART‑002 · Règle « Incomplet → Novice → À compléter »

Validé Type : Automatisation

Si le statut dossier d'un artisan est Incomplet et que son niveau devient Novice, alors le statut dossier passe automatiquement à À compléter.

DEVI‑001 · Pré‑requis ID devis pour « Devis envoyé »

Validé Type : Règle produit
  • L'ID du devis doit être renseigné avant le passage au statut « Devis envoyé ».
  • Si l'ID n'est pas renseigné, l'action « Passer à Devis envoyé » est masquée dans le clic droit.
  • Pas d'automatisation du passage « Demandé → Devis envoyé » lors de la saisie de l'ID.

À faire Type : Fonctionnel

Le contenu type du message donne lieu à une prévisualisation et à la possibilité de copier le message via une icône de copie.

DUP‑001 · « Devis supp » — duplication de dossier

À faire Type : Flux
  • L'action Devis supp crée une nouvelle ligne et un nouvel enregistrement dans la base de données (un devis = une ligne).
  • La duplication reprend les infos du dossier sauf : ID, Contexte, Consignes.
  • Sur le nouveau dossier, ajouter en commentaire : « devis supp avec l'ancien ID [ID] ».

ACPT‑001 · Acompte — saisie & réception

À faire Type : Automatisation
  1. Saisie du montant d'acompte réclamé
    Lorsqu'un utilisateur saisit le montant de l'acompte réclamé dans le CRM, le statut du dossier passe automatiquement à « Attente acompte » (att acompte).
    Condition : cette saisie n'est possible que si le dossier est actuellement au statut « Accepté ».
  2. Réception de l'acompte
    Lorsque l'acompte est effectivement reçu, le gestionnaire coche la case « Acompte reçu ».
    Cette action déclenche l'obligation de renseigner la date de réception de l'acompte.
    Tant que la date de réception n'est pas saisie, il est impossible de faire passer le dossier du statut « Attente acompte » à « Accepté ».
  3. Retour automatique au statut « Accepté $ »
    Une fois la date de réception saisie, le statut du dossier repasse automatiquement à « Accepté $ » (symbole $ affiché pour indiquer que l'acompte a été reçu).

Pour toute correction ou ajout, merci d'indiquer l'ID de la demande (ex. « DAT‑001 »).
